# Migraciones aplicadas

Registro manual de qué migraciones ya se corrieron en el proyecto real de Supabase (rios-system SQL Editor). Actualizar esta lista cada vez que se escriba y corra una migración nueva — evita no saber si una ya se aplicó o no.

- [x] `0001_init.sql` — esquema inicial: `profiles`, `categorias`, `productos`, `clientes`, `proveedores`, `cotizaciones`, `cotizacion_items`, `auditoria` + RLS.
- [x] `0002_productos_variantes_fotos.sql` — columnas `color`/`modelo`/`foto_url` en `productos` + bucket de Storage `productos` (público) con políticas de lectura pública / escritura autenticada.
- [x] `0003_productos_moneda_referencia.sql` — columnas `moneda` (PEN/USD, default PEN), `tipo_cambio` y `codigo_referencia` (único, independiente de `codigo_barras`) en `productos`.
- [x] `0004_cotizaciones_vencimiento.sql` — columna `vence_en` (trigger `before insert`, 48h desde `created_at`) en `cotizaciones` + función `expirar_cotizaciones_vencidas()` + cron job `pg_cron` cada 15 min que libera stock reservado y cancela cotizaciones sin aprobar vencidas.
- [x] `0005_cotizaciones_aprobar.sql` — columna `moneda` (PEN/USD, default PEN) en `cotizaciones` + función `aprobar_cotizacion()` (RPC): reserva stock atómicamente con bloqueo de filas (`for update`) y pasa la cotización de `enviada` a `reservada`, o revierte todo si falta stock.
- [x] `0006_cotizaciones_verificar_despacho.sql` — columnas `verificada_por`/`verificada_at` en `cotizaciones` + función `verificar_despacho_cotizacion()` (RPC, módulo Scanner): descuenta `stock_fisico` real según lo escaneado, libera `stock_reservado` según lo pedido, permite exceso, pasa la cotización de `reservada` a `lista_despacho` y deja el primer registro real en `auditoria`.
- [x] `0007_clientes_ruc_dni.sql` — columna `ruc_dni` en `clientes` (módulo Clientes), necesaria también para la futura facturación electrónica SUNAT en Ventas.
- [x] `0008_roles_admin_inicial.sql` — fija `onemillion0112@gmail.com` como `admin` + función helper `mi_rol()` (módulo Usuarios/Roles). Correr primero, antes que 0009/0010.
- [x] `0009_rls_por_rol.sql` — reemplaza las policies permisivas de `0001_init.sql` por RLS real según rol en `categorias`, `productos`, `clientes`, `cotizaciones`, `cotizacion_items`, `auditoria`; agrega trigger de protección de `rol` en `profiles` + RPC `cambiar_rol_usuario()`.
- [x] `0010_roles_en_rpcs_existentes.sql` — agrega chequeo de `mi_rol()` a `aprobar_cotizacion()` (admin/gerencia) y `verificar_despacho_cotizacion()` (almacen/admin).
- [x] `0011_productos_eliminar_solo_admin.sql` — restringe el borrado de `productos` a solo Admin (antes admin/gerencia); botón "Eliminar" nuevo en el módulo Productos.
- [x] `0012_proveedores_compras.sql` — módulo Proveedores + Compras: agrega `ruc`/`contacto`/`notas` a `proveedores` + RLS real (antes permisiva); tablas nuevas `compras`/`compra_items` con RLS (lectura abierta, alta Admin/Gerencia); función `recibir_compra()` (RPC): entrada atómica de `stock_fisico`, actualiza `precio_compra` y deja auditoría (Admin/Gerencia/Almacén).
- [x] `0013_compras_guia_remision.sql` — columna `guia_remision` (texto, referencia/trazabilidad) en `compras`; policy que permite editarla solo mientras la compra sigue `pendiente` (Admin/Gerencia/Almacén).
- [x] `0014_precio_mayorista_compra_items_edit.sql` — columna `precio_mayorista` en `productos` (el Excel del proveedor trae X MAYOR/PUBLICO); policies de update/delete en `compra_items` mientras la compra siga `pendiente` (Admin/Gerencia); `recibir_compra()` deja de pisar `precio_compra` cuando la línea tiene costo 0 ("costo aún no cargado").
- [x] `0015_inventario_kardex.sql` — módulo Inventario + Kardex: columna `stock_minimo` en `productos` + columna calculada `necesita_reposicion(productos)` (para filtrar "por reponer" server-side); tabla `movimientos_inventario` (ledger append-only del kardex, con RLS solo-lectura — lo escriben únicamente las RPCs security definer); retrofit de `recibir_compra()` (movimiento `entrada`) y `verificar_despacho_cotizacion()` (movimiento `salida`) para dejar su fila de kardex; RPC nueva `ajustar_stock()` (ajuste manual con motivo, Admin/Gerencia). El select de Productos ahora depende de `stock_minimo`.
- [x] `0016_ventas_caja.sql` — módulo Ventas internas + Caja: tablas `caja_sesiones` (con índice único parcial = una sola caja abierta a la vez), `ventas` (sin campos tributarios — el comprobante va separado para SUNAT), `venta_items` y `venta_pagos` (pago mixto); RLS solo-lectura (todo se escribe por RPC). RPCs `abrir_caja`/`cerrar_caja` (arqueo de efectivo), `registrar_venta` (atómica: valida caja abierta + stock_disponible, descuenta stock_fisico, deja movimiento `salida` en el kardex, crea venta+items+pagos) y `anular_venta` (Admin/Gerencia; devuelve stock con movimiento `entrada`). Todas restringidas por `mi_rol()` (vender: admin/gerencia/ventas).
- [x] `0017_reportes.sql` — módulo Reportes (BI): 5 RPCs de agregación **solo Admin/Gerencia** (usan `precio_compra`, calculan en la base): `reporte_ventas` (total/margen/ticket/serie por día/por método/por vendedor), `reporte_top_productos`, `reporte_valorizacion_inventario` (capital en stock por marca), `reporte_stock_inmovilizado` (productos con stock sin venta en N días) y `reporte_ventas_por_modelo` (vertical del rubro moto). Solo crea funciones (no toca tablas); días agrupados en hora de Lima, montos en PEN.
- [x] `0018_configuracion_guias.sql` — módulo Configuración de empresa + Guías de remisión (fase 5): tabla `configuracion_empresa` (fila única, sembrada con los datos reales de la empresa: razón social, RUC, domicilios, teléfonos + logo_url/marcas_url) editable por Admin/Gerencia; bucket de Storage `empresa` (público, escritura Admin/Gerencia) para logo y tira de marcas; tabla `documento_correlativos` (numeración atómica por tipo/serie); tablas `guias_remision` + `guia_remision_items` (guía remitente imprimible, snapshot del destinatario, RLS solo-lectura + anular Admin/Gerencia); RPC `crear_guia_remision` (atómica, toma correlativo + crea guía+items, roles admin/gerencia/ventas/almacén). El logo/marcas se suben desde la pantalla Configuración.
